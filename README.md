## Project overview
This project implements a backend for a supermarket system exposing REST APIs to manage:
- Categories
- Products
- Customers
- Orders and order items

It uses DTOs, services, repositories and controllers with JPA entities persisted to a relational database. Basic validation and global exception handling are included.

---

## Tech stack
- Java 17 (project target)
- Spring Boot 3.1.4 (web, data-jpa, validation)
- Spring Data JPA / Hibernate
- H2 (in-memory DB for local dev & tests by default)
- (optional) PostgreSQL — you can switch to this in `application.yml` or via environment variables
- Lombok (annotation-based getters/setters/builders)
- ModelMapper (DTO ↔ entity mapping)
- Maven (build tool)
- JUnit + Mockito (unit tests)

## Where things live (high-level)
- `src/main/java/com/example/supermarket/controller/` — REST controllers for each resource (CategoryController, ProductController, CustomerController, OrderController)
- `src/main/java/com/example/supermarket/service/` and `.../service/impl/` — service interfaces and implementations (business logic, transactions)
- `src/main/java/com/example/supermarket/repository/` — Spring Data JPA repositories for CRUD
- `src/main/java/com/example/supermarket/entity/` — JPA entity classes (Category, Product, Customer, Order, OrderItem)
- `src/main/java/com/example/supermarket/dto/` — DTOs used by controllers (CategoryDto, ProductDto, CustomerDto, OrderDto, OrderItemDto)
- `src/main/java/com/example/supermarket/config/` — ModelMapper and initial data loader
- `src/main/resources/schema.sql` and `data.sql` — DB initialization SQL (if `spring.sql.init.mode` is enabled)
- `src/main/resources/application.yml` — application configuration including datasource

## Database configuration
By default the project uses an in-memory H2 database for development and tests. The relevant configuration is in `src/main/resources/application.yml` (excerpt):

```yaml
server:
  port: 8080
spring:
  datasource:
    url: jdbc:h2:mem:supermarketdb;DB_CLOSE_DELAY=-1;DB_CLOSE_ON_EXIT=FALSE
    driver-class-name: org.h2.Driver
    username: sa
    password: password
  jpa:
    hibernate:
      ddl-auto: none
    show-sql: true
    defer-datasource-initialization: true
  sql:
    init:
      mode: always
  h2:
    console:
      enabled: true
      path: /h2-console
```

Notes:
- `defer-datasource-initialization: true` ensures Spring Boot initializes the datasource first and then runs `schema.sql` and `data.sql` during startup.
- `schema.sql` creates tables; `data.sql` inserts sample rows. They live at `src/main/resources`.

Switching to PostgreSQL (example)
1. Add PostgreSQL dependency if not already present (pom already uses Spring Boot starters; add `org.postgresql:postgresql` if you want runtime PG driver).
2. Update `application.yml` (or create `application-prod.yml`) with PostgreSQL connection, e.g.:

```yaml
spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/supermarketdb
    driver-class-name: org.postgresql.Driver
    username: postgres
    password: mysecret
```

3. Start your Postgres server and create the database `supermarketdb` if not present:

```sql
CREATE DATABASE supermarketdb;
```

4. Run the app and check tables in pgAdmin.

Important: If switching DB engines, ensure the SQL in `schema.sql` uses compatible identity/auto-increment syntax for your DB (the repo uses H2-friendly `AUTO_INCREMENT` values). For PostgreSQL you may prefer `GENERATED BY DEFAULT AS IDENTITY` or SERIAL depending on PG version.

## Useful SQL queries (SELECT / ORDER BY / INSERT)
Copy/paste these into pgAdmin, H2 Console, or any SQL client connected to your DB.

A) View data ordered by name
- Categories ordered by name
```sql
SELECT * FROM categories ORDER BY name ASC;
```
- Products ordered by name
```sql
SELECT * FROM products ORDER BY name ASC;
```
- Customers ordered by first name then last name
```sql
SELECT * FROM customers ORDER BY first_name ASC, last_name ASC;
```

B) Orders ordered by customer name (join)
```sql
SELECT o.*, c.first_name, c.last_name
FROM orders o
JOIN customers c ON o.customer_id = c.id
ORDER BY c.first_name ASC, c.last_name ASC, o.order_date DESC;
```

C) Order items ordered by product name (join)
```sql
SELECT oi.*, p.name AS product_name
FROM order_items oi
JOIN products p ON oi.product_id = p.id
ORDER BY p.name ASC, oi.id ASC;
```

D) Products grouped by category name
```sql
SELECT p.*, c.name AS category_name
FROM products p
LEFT JOIN categories c ON p.category_id = c.id
ORDER BY c.name ASC NULLS LAST, p.name ASC;
```

E) Quick INSERT sample set (creates 3 categories, 3 customers, 6 products, 3 orders, and order items)
Paste in order (categories → customers → products → orders → order_items) so FK constraints succeed.


## Postman and curl examples
Use the base URL `http://localhost:8080` (default port 8080). Add header `Content-Type: application/json` for POST and PUT.

1) Create Category
- POST http://localhost:8080/api/categories
Body:
```json
{ "name": "Beverages", "description": "Drinks and cold beverages" }
```

2) Create Product
- POST http://localhost:8080/api/products
Body:
```json
{
  "name": "Cola",
  "sku": "COLA-001",
  "description": "Cold drink",
  "price": 1.50,
  "stock": 100,
  "categoryId": 1
}
```

3) Create Customer
- POST http://localhost:8080/api/customers
Body:
```json
{
  "firstName": "John",
  "lastName": "Doe",
  "email": "john.doe@example.com",
  "phone": "1234567890",
  "address": "123 Main St"
}
```

4) Create Order
- POST http://localhost:8080/api/orders
Body:
```json
{
  "customerId": 1,
  "items": [
    { "productId": 1, "quantity": 1 },
    { "productId": 3, "quantity": 2 }
  ]
}
```

cURL example to create a customer:

```bash
curl -X POST http://localhost:8080/api/customers \
  -H "Content-Type: application/json" \
  -d '{"firstName":"Jane","lastName":"Smith","email":"jane.smith@example.com","phone":"9876543210","address":"456 Other St"}'
```

From project root (requires Maven & JDK 17):


## Troubleshooting (common issues)
1. Lombok / javac mismatch (NoSuchFieldError referencing com.sun.tools.javac)
   - Ensure your IDE and Maven use JDK 17 for compilation/annotation processing.
   - In IntelliJ: set Project SDK to Java 17, enable annotation processing (Settings → Compiler → Annotation Processors), and set Maven importer/runner to JDK 17.
   - You can run Maven in PowerShell with a JDK 17 environment:

```powershell
$env:JAVA_HOME='C:\Program Files\Java\jdk-17'
$env:Path="$env:JAVA_HOME\bin;$env:Path"
mvn -v
mvn -U clean package
```

2. SQL initialization errors
   - If you get errors when Spring runs `schema.sql`, check the SQL dialect compatibility (H2 vs Postgres). Adjust `schema.sql` to match the DB engine you use.

3. If you see validation errors (400) from the API, the response body contains messages from the `@Valid` annotations on DTOs (e.g., missing required fields, invalid email, price must be > 0).
